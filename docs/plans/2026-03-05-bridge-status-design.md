# 设计文档：Bridge 状态回显

**日期**：2026-03-05
**状态**：已批准，待实施

---

## 设计目标

发送方在消息发出后，能以非评判的方式看到 AI 正在为对方建立理解桥梁，从而感受到"被支持"而非"被审视"，降低因等待对方回应而产生的焦虑。

---

## 核心决策

| 问题 | 决策 |
|---|---|
| 发送前预警？ | 否，打断输入节奏，且发后看到也能消气 |
| 发送方看 AI 分析？| 是，但用"桥梁框架"（Framework Y）呈现 |
| 文化上下文注释？ | 否，AI 分析本身已覆盖 |
| 额外 API 调用？ | 否，复用已有 `insight` 字段 |

---

## 功能设计

### 触发条件

`ai-analysis` socket 事件中，`analysis.senderId === myId`（当前用户是发送方）。

### 视觉呈现

在发送方消息气泡行下方插入一个状态行：

```
💡 Bridge 已建立 · [insight 截断至 30 字]
```

**样式规格：**
- 位置：插入到 `.msg-row.mine` 之后，作为兄弟元素
- 字体：`0.65rem`
- 颜色：`rgba(255,213,79,0.5)`（淡金）
- 出现：`opacity: 0 → 1`，transition 0.4s
- 3 秒后降至 `opacity: 0.3`（持续可见但退至背景）
- class：`.bridge-status`

### 文案格式

```
💡 Bridge 已建立 · [insight 前 30 字符]…
```

`insight` 字段已是接收方视角的共情描述，发送方看到后读作"这是 AI 帮助对方理解我的方式"，符合 Framework Y。
